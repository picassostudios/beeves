# beavus — web shell

A thin TypeScript + Vite shell over the Rust/WASM `WasmApp`. All editing
logic lives in Rust; this package only renders the canvas and forwards
input events.

## Prerequisites

- Node.js 18+ and npm
- A WebGPU-capable browser (recent Chrome/Edge, or Safari Technology Preview)
- The Rust toolchain, [`wasm-pack`](https://rustwasm.github.io/wasm-pack/), the wasm
  target (`rustup target add wasm32-unknown-unknown`), and
  [`cargo-watch`](https://crates.io/crates/cargo-watch) (`cargo install cargo-watch`) for
  live reload on Rust changes

## Run the dev server

```sh
cd web
npm install
npm run dev
```

Open the printed URL (default http://localhost:5173). **The WASM builds automatically and
live-reloads on Rust changes:**

- A `predev` hook runs `wasm-pack build --dev` (→ `web/pkg/`) before Vite starts, so the
  package always exists.
- `npm run dev` then runs two processes concurrently (via `concurrently`): Vite **and**
  `cargo watch`, which watches all three Rust crates (`app_core`, `renderer`, `app_wasm`)
  plus the WGSL shaders and rebuilds the WASM on any change.
- A small Vite plugin (`reloadOnWasm` in `vite.config.ts`) watches `web/pkg/` and triggers
  a full browser reload after each rebuild — so editing Rust and saving auto-refreshes the
  page (typically ~1–2 s for an incremental rebuild). No restart needed.

Requires `cargo-watch` (`cargo install cargo-watch`).

## Scripts

```sh
npm run dev         # predev builds wasm, then runs vite + cargo-watch concurrently (live reload)
npm run build       # prebuild: wasm-pack build (release) -> web/pkg, then tsc && vite build
npm run wasm        # build the optimized (release) WASM package only
npm run wasm:dev    # build the debug WASM package only (fast, no wasm-opt)
npm run watch:wasm  # cargo-watch the Rust crates -> rebuild debug WASM on change
npm run typecheck   # tsc --noEmit  (needs web/pkg; run `npm run wasm:dev` first)
npm run preview     # serve the production build
```

`web/pkg/` is generated by `wasm-pack` (git-ignored). The dev/build scripts create it
automatically; only standalone `typecheck` needs it built first.

## Toolbar

- **Brush** — drag to draw a polyline; on release it is fit to a Bezier
  skeleton and added as a stroke.
- **Bezier** — drag the nearest control handle.
- **Sculpt** — drag to push splats on the nearest stroke (uses the Size
  slider as the sculpt radius).
- **Select** — click to hit-test/select a splat.

**Navigating** needs no tool: a trackpad two-finger swipe (or a scroll wheel)
pans the canvas, and **Shift+scroll** — or a trackpad pinch — zooms at the
cursor.

Use the color swatch and size slider to set brush parameters. **Save**
downloads the document as `*.gspf.json`; **Load** reads one back in. The
status readout shows the live splat count.

## Layout

```
web/
  index.html            canvas + toolbar + minimal CSS
  vite.config.ts        wasm + top-level-await plugins, `pkg` alias
  tsconfig.json         strict, bundler resolution, ES2022 + DOM
  src/
    main.ts             init wasm, wire DOM events, render loop
    wasm.ts             typed loader (init + WasmApp)
    tools/toolbar.ts    toolbar -> set_tool wiring
  pkg/                  generated by wasm-pack predev/prebuild hook (git-ignored)
```
